# Caves of Ooo ASCII World Rendering

## Canonical World Render Contract
- World rendering is ASCII-only for now.
- The authoritative world render fields are:
  - `Render.DisplayName`
  - `Render.RenderString`
  - `Render.ColorString`
  - `Render.RenderLayer`
- `Render.RenderString` must be exactly one printable glyph.
- `Render.ColorString` is the authoritative foreground tint.
- `Render.Tile`, `Render.TileColor`, and `Render.DetailColor` are retained only for compatibility/imported data. The world renderer ignores them.

## Runtime Path
1. `Assets/Scripts/Data/Factories/EntityFactory.cs`
   - Loads and bakes blueprints.
   - Validates world-facing ASCII render metadata through `ValidateAsciiWorldBlueprints()`.
2. `Assets/Scripts/Presentation/Bootstrap/GameBootstrap.cs`
   - Runs the ASCII validation pass immediately after blueprint load.
3. `Assets/Scripts/Presentation/Rendering/AsciiWorldRenderPolicy.cs`
   - Defines the canonical glyph and color validation/fallback rules.
4. `Assets/Scripts/Presentation/Rendering/ZoneRenderer.cs`
   - Chooses the cell's top visible entity by `RenderLayer`.
   - Resolves the displayed glyph from `RenderString`.
   - Resolves the foreground tint from `ColorString`.
   - Applies render-event color mutation.
   - Renders blank cells as blank, not fake floor dots.

## Content Rules
- Terrain must be explicit. Open ground should come from a real terrain entity in the cell.
- Standard world glyph policy:
  - floors: `.`, `,`, `;`
  - walls: `#`
  - water: `~`
  - trees: `T`
  - vegetation: `;`, `"`, `%`, `^`
  - player: `@`
  - stairs: `>` and `<`

## Generation Rules
- `Assets/Scripts/Gameplay/World/Generation/Builders/CaveBuilder.cs`
  - Open cave cells now always receive `Floor` or `Rubble`.
- `Assets/Scripts/Gameplay/World/Generation/Builders/JungleBuilder.cs`
  - Open jungle cells now always receive `Grass`.
- The renderer no longer hides generation holes by painting fallback floor dots.

## Asset Boundary
- The world renderer does not use `Assets/Resources/QudTextures` or any sprite/tile art directory for world drawing.
- World drawing uses the runtime CP437 atlas generated by:
  - `Assets/Scripts/Presentation/Rendering/CP437TilesetGenerator.cs`
- UI remains on the existing glyph/tilemap path; this migration is world-first only.

## Camera Policy
- Normal gameplay zoom is driven by a target visible tile count in:
  - `Assets/Scripts/Presentation/Cameras/CameraFollow.cs`
- Default vertical target is `34` tiles.
