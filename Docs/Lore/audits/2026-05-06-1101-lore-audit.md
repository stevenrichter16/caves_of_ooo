## 5. Anti-style self-check

**Quoted sentence (from §3):**

> *"The Choir cannot differentiate by reputation not because the
> system can't, but because no choice in `RotChoir.json` declares
> the predicate."*

This reads as a code-review observation, not a TES book. *The
system* is a software register, not a mythologised one (no *the
System*, no *Holy Engine*). *Declares the predicate* is plain
programming language. The clause structure — *not because X, but
because Y* — is everyday explanatory grammar. No archaic verbs,
no honorifics, no apostrophed proper nouns, no ornament. The
sentence states a fact about a JSON file using the JSON file's
own vocabulary.

TES rewrite (forbidden): *"And so it is, beloved walker, that
the Choir's tongue is bound — not by want of ancient art, but
by the silent absence of the binding-rune within the holy
parchment of Rot-Speech."* Tells: *beloved walker*; *bound*,
*holy*, *Rot-Speech* allegorically capitalised; pseudo-archaic
inversion *And so it is*. The actual sentence is technical, flat,
no ornament. The reasoning is the rhetoric.

Pass.
