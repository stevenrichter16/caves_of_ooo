### 2.1 🟡 The 0805 §3 hygiene-item-4 check command is wrong for the actual procedure

**File:** `2026-05-06-0805-lore-audit.md` §3 *The check, in
shell-form.*

The 0805 proposed:

```sh
git -C /tmp/routine-lore-wt status --short Docs/Lore/audits/
ls -t Docs/Lore/audits/*.md | head -3
```

Two problems with the first command. (a) If `/tmp/routine-lore-wt`
does not yet exist (the worktree has not been added this
sitting), `git -C` errors and the audit-start mistakes the error
for *no parallel firing.* (b) The command checks the worktree's
working-tree status, but a parallel firing's uncommitted draft
would be in the *parallel* worktree, not in this one. The check
needs to inspect the main repo's worktree list and any
*independently-existing* `routine-lore` worktrees.

**Correction (working procedure for hygiene item 4):**

```sh
# Run before creating this sitting's worktree.
git -C /Users/steven/house_feature/caves_of_ooo worktree list \
    | grep routine-lore
# If a worktree path is returned, check it for uncommitted .md:
git -C <returned-worktree-path> status --short Docs/Lore/audits/
```

The 0901 ran this corrected check at audit-start (see header
block); it returned empty, confirming no parallel firing. The
0805's command would have errored, then incorrectly concluded
*no parallel.* Failure mode would have been silent.

**Severity 🟡.** Future hygiene-item-4 maintenance should
codify the corrected command. Sitting 23 (or whichever next
firing engages hygiene item 4) inherits the corrected procedure.

### 2.2 🔵 The 0805 §4 standing-flag overdue threshold is underspecified

**File:** `2026-05-06-0805-lore-audit.md` §4 *What unblocks the
next fresh sitting.*

The 0805 named *standing-flag overdue* as a third unblocker
beyond bible-mtime-moves and JSON-patches-shipped, citing item
31 (Glassblown-Choir custody) as flagged across four sittings
(0407 / 0501 / 0607 / 0701). But the 0805 did not specify a
**threshold** — at what flag-count does a candidate become
overdue? Four (item 31's current count)? Five? Eight?

Without a threshold, the unblocker can fire whenever the
audit chooses. That is exactly the candidate-producing
behaviour the hold-discipline is meant to prevent. The
unblocker is currently a soft permission to break the hold,
not a hard rule.

**Severity 🔵.** Two ways to tighten:

  - **Hard rule (recommended):** standing-flag becomes overdue
    at **eight sittings** of standing-flag. This is roughly
    twice the current item 31 count (4 sittings → 8). The
    chain has produced ~10 sittings per 24 hours at recent
    rate; eight sittings is just under one calendar day of
    standing-flag.
  - **Conditional rule:** standing-flag becomes overdue when
    **the candidate's working-content has been independently
    referenced by three other audits' content** (i.e., the
    candidate has cross-faction implications other audits
    have leaned on). Item 31's current cross-references: the
    0407 §3 named Glassblown-Choir as the *un-named seam* the
    chain had identified; the 0501 §3 mentioned Glassblown's
    silence in the JSON `OtherVersions`; the 0701's §3
    Thermoclave consolidation pulled threads that touched
    Glassblown-Choir. Three cross-references; meets the
    conditional rule.

The audit prefers the **hard rule** (eight sittings) for
machine-readability. Item 31 currently sits at four sittings
of standing-flag; under the hard rule, it would not become
overdue until ~four more sittings. The hold continues.

## 5. Anti-style self-check

**Quoted sentence (from §4 retrospective):**

> *"What an audit cannot do is press Save."*

Working register, gloss-form. *Save* is the working clerical
verb the editor uses about the working-action that commits a
file change to disk — not figurative, not heraldic. *Press* is
the working verb for *physical actuation of a key.* The
sentence's claim is procedural about the audit's own scope —
the chain's tool is *propose,* not *commit.* Lower-case
throughout. No allegorical capitalisation. The proverb-shape
is in working-register English, not in mythic-sentence cadence.

TES rewrite (forbidden): *"That which the Watcher cannot do is
to lift the Pen — for the Pen is the Editor's, and the Watcher's
hand falls short of it, by the long Custom of the Inscriptions."*
Tells: *Watcher*, *Pen*, *Editor*, *Custom*, *Inscriptions*
(allegorical capitalisations); *That which the Watcher cannot
do* (mythic-relative-clause); *by the long Custom* (KJV-archaic
ablative). The actual sentence: *audit*, *cannot*, *press*,
*Save* — four working-register terms, the proverb-shape
(*what X cannot do is Y*) doing the rhetorical work.

Pass.
